### Real bugs found only by actually running the app (via Detox), not by review

1. **Crash on first camera-preview render (iOS):** `CircularBufferPreviewView.isActive` wasn't `@objc dynamic`, so `RCT_EXPORT_VIEW_PROPERTY`'s Objective-C KVC bridge couldn't find a setter — `unrecognized selector sent to instance` on every real launch. Fixed.
2. **All `react-native-permissions` calls silently broken (iOS):** the Podfile never called `setup_permissions([...])`, so zero permission handlers were compiled in — `check()`/`request()` failed at runtime with "No permission handler detected" for every permission, every time. Fixed.
3. **iOS app had likely never been compiled before this audit:** the `CircularBuffer` Xcode group's path was `../CircularBuffer` (one directory too high), so every one of the 7 native buffer files failed to resolve on build. Fixed.

## Summary of what was broken and is now fixed

- **iOS couldn't build at all** (wrong Xcode group path) — likely never compiled before this session.
- **iOS couldn't `pod install`** (Picovoice voice-processor pod incompatible with RN 0.86's new-arch packaging) — patched via `patch-package`.
- **The camera preview crashed the app on real launch** (missing `@objc dynamic`).
- **Every iOS permission check silently failed** (missing `setup_permissions` in the Podfile).
- **A camera-busy/session error never reached the JS UI** (no event emitter wiring) — now it does.
- **A wake-word engine failure banner flashed and vanished instantly** (self-retrigger bug) — now it persists correctly.
- **Jest, RNTL, and `yarn lint` had never actually been run** — all now pass cleanly (11 suites / 58 tests).
- **No iOS unit test target existed** — created and passing.
